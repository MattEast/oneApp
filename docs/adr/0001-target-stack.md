# ADR 0001: Target Stack

## Status
Accepted

## Context
The product vision requires secure identity, financial data management, reporting, notifications, and strong automated testing. The current prototype stack is not the intended long-term architecture.

## Decision
Adopt the target stack defined in `docs/TECH_STACK.md` as the basis for future implementation planning.

## Consequences
- Planning and delivery documentation should align to the target stack.
- Prototype implementation details should not drive long-term architecture decisions.
- Migration and realignment work should be explicit in roadmap artifacts.
